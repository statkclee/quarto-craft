Read README.md in this folder for the full content + visual foundations, then explore the other files listed below.

1.2 Workflow

If creating visual artifacts (slides, mocks, throwaway prototypes, marketing pages): 1. Read README.md for tone + foundations. 2. Copy colors_and_type.css and assets/ into your new project. 3. Reference patterns from ui_kits/ and slides/ — those mirror the canonical visual style. 4. Produce a static HTML file the user can view. Keep lang="ko", word-break: keep-all, and line-height: 1.75 on <body>.

If working on production Quarto code: 1. Use the actual SCSS sources from design-system/ and scss/ — don’t hand-translate the CSS. 2. Set format.html.theme: [cosmo, scss/quarto-theme.scss] in _quarto.yml. 3. Use @use / @forward to extend. Never @import SCSS tokens.

1.3 Brand rules (TL;DR)

  • Brand colour: #4F46E5 (indigo-600). Single ramp, no other hues used decoratively.
  • Body font: Pretendard Variable, 16px, line-height 1.75, word-break: keep-all.
  • Heading font: same (Pretendard). Serif (Noto Serif KR) reserved for blockquotes.
  • Mono: JetBrains Mono.
  • Cards: white bg, 1px neutral-200 border, 8px radius, shadow-sm. Never coloured fills.
  • Callouts: 4px coloured left border, 5% colour tint, semantic colour for header text.
  • Gradients: only on title blocks / slide title backgrounds. Always primary-600 → primary-400.
  • Icons: Bootstrap Icons line variants. No custom SVG illustrations.
  • Korean-first: lang=“ko” by default; allow English technical terms inline. Polite-neutral register, no second person, no decorative emoji.

1.4 When unsure

If the user asks for something the system doesn’t directly cover (a colour outside the ramp, a new icon style, a non-Korean layout), default to the closest neighbour in the existing system and flag the substitution. Don’t invent new visual primitives without permission.
