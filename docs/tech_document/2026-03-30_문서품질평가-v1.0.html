1 Quarto Craft v1.0 문서 품질 평가

  • 평가일: 2026-03-30
  • 평가 대상: docs/ 하위 5개 템플릿 렌더링 결과
  • 평가 방법: 브라우저 스크린샷 + PDF 직접 확인

1.1 종합 점수

영역 점수 등급
A. 타이포그래피 7/10 양호
B. 색상 & 시각 디자인 5/10 보통
C. 레이아웃 & 구조 6/10 보통
D. 컴포넌트 품질 6/10 보통
E. 템플릿별 완성도 5/10 미흡
총점 29/50 보통

1.2 A. 타이포그래피 (7/10)

1.2.1 잘된 점

  • Pretendard 웹폰트 정상 로딩, 한국어 렌더링 깔끔
  • 제목 계층 (H1~H3) 크기 차이 명확
  • 인용문 Noto Serif KR 서체 전환 작동
  • word-break: keep-all 한국어 줄바꿈 정상 적용
  • 코드 블록 JetBrains Mono 구분 명확

1.2.2 문제점

  1. H1 “보고서 제목”이 너무 밋밋함 — 볼드 텍스트만 있고 시각적 임팩트 부족
  2. 제목과 부제목 간격이 좁음 — 계층 구분이 약함
  3. 본문 행간이 실제로 적용되었으나 체감 차이 미미 — Quarto 기본 스타일과의 충돌 가능성

1.2.3 v2.0 개선안

  • 타이틀 블록에 배경색/하단 구분선 추가
  • 부제목 색상 차별화 (secondary 색상)
  • H2 하단에 얇은 구분선 추가로 섹션 구분 강화

1.3 B. 색상 & 시각 디자인 (5/10)

1.3.1 잘된 점

  • 콜아웃 3종 (note=파랑, tip=초록, warning=주황) 색상 구분 정상
  • 링크 색상 인디고 적용

1.3.2 문제점

  1. 전체적으로 흰 배경 + 검은 텍스트만 있어 밋밋함 — 디자인 토큰이 시각적으로 드러나지 않음
  2. 타이틀 영역에 브랜드 색상 없음 — primary 인디고가 제목이나 메타데이터에 미사용
  3. 코드 블록 배경이 거의 흰색 — $color-bg-code: #F8F9FC가 너무 연해서 본문과 구분 안 됨
  4. 표 헤더가 uppercase로 안 됨 — Quarto 기본 스타일이 오버라이드
  5. Dashboard valuebox 색상이 디자인 토큰 무시 — Bootstrap 기본 primary/success/info 색상 사용

1.3.3 v2.0 개선안

  • 타이틀 블록에 primary-50 배경 + primary-600 하단 라인
  • 코드 블록 배경색 더 진하게 (#F1F3F8)
  • H2 하단에 primary-200 그라데이션 구분선
  • 표 헤더 배경색 추가 (neutral-50)

1.4 C. 레이아웃 & 구조 (6/10)

1.4.1 잘된 점

  • TOC 우측 고정 표시 정상
  • 번호 매기기 (1, 1.1, 2, 2.1…) 정상
  • 마진 참조(reference-location: margin) 작동

1.4.2 문제점

  1. 타이틀 블록이 본문과 시각적으로 분리 안 됨 — abstract, 메타데이터, 본문이 한 덩어리
  2. abstract “초록” 레이블이 너무 작고 밋밋 — 시맨틱 구분 약함
  3. Book 템플릿 번호 체계 어색 — “0.1 이 책에 대하여” (서문이 0번)
  4. 섹션 간 간격이 균일하지 않음 — H2와 다음 H2 사이 공간이 부족

1.4.3 v2.0 개선안

  • abstract 영역에 배경색 + 보더 스타일링
  • 메타데이터 블록 시각적 정리 (아이콘 or 구분선)
  • Book 서문에 .unnumbered 적용 확인
  • 섹션 간 상단 마진 확대 (margin-top: 3rem)

1.5 D. 컴포넌트 품질 (6/10)

1.5.1 잘된 점

  • 콜아웃 note/tip/warning 3종 시각적 구분 명확
  • 왼쪽 4px 보더 + 배경 투명도 5% 조합 깔끔
  • 표 줄무늬 배경 (even row) 작동
  • 코드 블록 구문 강조 (syntax highlighting) 정상

1.5.2 문제점

  1. 인라인 코드 보더가 너무 강함 — 1px solid 보더가 텍스트 흐름을 끊음
  2. 콜아웃 헤더 아이콘이 Quarto 기본 아이콘 — 커스텀 스타일과 불일치
  3. 표 캡션 위치가 하단 왼쪽 — “타이포그래피 토큰 요약” 텍스트가 눈에 안 띔
  4. 코드 블록과 본문 간 간격 불균일

1.5.3 v2.0 개선안

  • 인라인 코드 보더 제거, 배경색만 유지
  • 콜아웃 전체적으로 border-radius 조정
  • 표 캡션 스타일 강화 (font-weight: medium, 이탤릭)

1.6 E. 템플릿별 완성도 (5/10)

1.6.1 Report (7/10)

  • 구조적으로 안정적, 콘텐츠 샘플 충분
  • 타이틀 블록 디자인 개선 필요

1.6.2 Presentation (5/10)

  • 타이틀 슬라이드 깔끔하나 너무 단순
  • 컨텐츠 슬라이드가 텍스트만 있어 밋밋 — 배경, 진행 바, 브랜딩 요소 없음
  • 슬라이드 하단 인디고 라인이 유일한 브랜드 요소

1.6.3 Dashboard (3/10)

  • 파이 차트에 matplotlib 객체 정보가 텍스트로 출력 — <matplotlib.patches.Wedge at 0x...> 노출
  • Y축 레이블 ’방문자 수’가 90도 회전되어 읽기 어려움
  • valuebox 색상이 디자인 토큰과 무관한 Bootstrap 기본값
  • 라인차트의 fill 영역이 기본 파란색

1.6.4 Book (6/10)

  • 좌측 챕터 네비게이션 정상 작동
  • 검색 기능 포함
  • “0.1 이 책에 대하여” 번호가 어색
  • 컨텐츠 양이 적어 페이지가 빈 느낌

1.6.5 Typst PDF (7/10)

  • Pretendard 폰트 정상 적용 (3페이지)
  • 목차 자동 생성 깔끔
  • 코드 블록 구문 강조 작동
  • 수식 렌더링 완벽
  • 표 스타일 기본적이지만 깔끔
  • 타이틀 페이지 디자인 없음 — 제목+저자+날짜만 중앙 정렬

1.7 v2.0 핵심 개선 목록

1.7.1 Priority 1 (필수)

  1. 타이틀 블록 리디자인 — 배경색, 구분선, 브랜드 컬러 적용
  2. Dashboard matplotlib 코드 수정 — 객체 출력 제거
  3. 인라인 코드 보더 제거 — 배경색만 유지
  4. 코드 블록 배경색 강화 — 본문과 구분 강화

1.7.2 Priority 2 (개선)

  1. H2 하단 구분선 — 섹션 시각적 분리
  2. abstract 영역 스타일링 — 배경 + 보더
  3. 표 헤더 배경색 추가
  4. Presentation 브랜딩 — 슬라이드 푸터, 진행 바

1.7.3 Priority 3 (마무리)

  1. Book 번호 체계 수정
  2. 섹션 간 간격 확대
  3. 표 캡션 스타일 개선
  4. Typst PDF 타이틀 페이지 디자인
