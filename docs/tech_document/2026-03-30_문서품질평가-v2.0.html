1.2 잘된 점 (v1.0 대비 개선 확인)

1.2.1 1. 타이틀 블록 — 큰 폭 개선

  • 인디고 그라데이션 배경 + 하단 보더로 시각적 임팩트 확보
  • 부제목이 primary-700 색상으로 분리되어 계층 구분 명확
  • Book 템플릿에도 동일하게 적용됨 — 일관성 유지

1.2.2 2. 섹션 구분 — H2 하단 구분선

  • primary-100 색상의 2px 구분선이 섹션 분리에 효과적
  • 섹션 간 상단 마진 3rem으로 확대되어 여백 충분
  • Presentation에서도 H2 구분선이 슬라이드 제목-본문 사이 구분자로 작동

1.2.3 3. 인라인 코드 — 개선 완료

  • 보더 제거로 텍스트 흐름이 자연스러워짐
  • 배경색 #EEF0F6가 본문과 적절히 구분됨

1.2.4 4. 코드 블록 — 배경 강화

  • #F4F5FA 배경으로 본문과 확실히 구분
  • 구문 강조가 배경과 잘 어울림

1.2.5 5. 표 — 헤더 개선

  • 헤더에 neutral-50 배경 + primary-200 하단 보더 적용
  • 캡션이 이탤릭 + medium weight로 눈에 더 잘 띔

1.2.6 6. Dashboard — 차트 정상화

  • 라인차트: 그리드, 마커 스타일 개선
  • 파이차트 → 도넛차트 전환으로 현대적 느낌
  • matplotlib 객체 텍스트 출력 완전 제거

1.2.7 7. Book — 번호 수정

  • “0.1 이 책에 대하여” → “이 책에 대하여”로 자연스러운 번호 제거

1.3 개선이 필요한 점

1.3.1 P1. Presentation 브랜딩 부재 (가장 시급)

  • 문제: 타이틀 슬라이드가 흰 배경 + 검은 텍스트만. 브랜드 정체성 없음
  • 현상: 모든 슬라이드가 기본 RevealJS 흰색 배경, 하단 인디고 라인만 유일한 컬러
  • 마지막 슬라이드 “감사합니다”: 왼쪽 정렬 + .center 클래스 미작동
  • 개선안: 타이틀 슬라이드에 primary 배경색, 슬라이드 하단 푸터/진행 바, 마지막 슬라이드 중앙 정렬

1.3.2 P2. 메타데이터 영역 (저자/공개) 부유감

  • 문제: 타이틀 블록과 abstract 사이에 “저자 / 작성자 | 공개 / 2026년 03월 30일”이 시각적으로 고립
  • 현상: 배경도 보더도 없이 떠 있음
  • 개선안: 메타데이터를 타이틀 블록 내부로 이동하거나, 경량 배경/보더 추가

1.3.3 P3. Report 타이틀-메타-abstract 간격

  • 문제: 타이틀 블록 하단과 메타데이터 사이 공백이 과다 (약 40px)
  • 개선안: 타이틀 블록 margin-bottom 축소 또는 메타데이터 margin-top 축소

1.3.4 P4. 콜아웃 border-radius 불일치

  • 문제: 콜아웃 외부는 radius-md이지만, 내부 헤더 배경이 왼쪽만 라운드되어 어색
  • 개선안: 콜아웃 헤더 배경에 상단 radius 적용

1.3.5 P5. TOC 항목 간격

  • 문제: TOC에서 “2 타이포그래피 샘플” 아래 하위 항목(2.1, 2.2…) 간격이 좁음
  • 개선안: TOC li 간격 미세 확대, depth별 들여쓰기 확인

1.3.6 P6. 코드 블록 하단 수평선 겹침

  • 문제: 코드 블록 바로 아래 회색 수평선(hr)이 코드 블록 보더와 시각적 중복
  • 현상: Presentation 코드 슬라이드에서 특히 뚜렷
  • 개선안: 코드 블록 직후 hr에 추가 마진 또는 투명도 조정

1.3.7 P7. Dashboard valuebox 색상

  • 문제: valuebox가 Bootstrap 기본 primary(파랑)/success(초록)/info(보라) 사용
  • 개선안: 디자인 토큰의 $color-primary-500 등을 적용하도록 CSS 오버라이드
