# Typst PDF 레이아웃 고도화 — L4 `_brand.yml` 통합

- **작업일**: 2026-05-16
- **범위**: 루트 `_brand.yml` 추가, 8개 템플릿 정리, Typst·HTML 양 포맷에 브랜드 토큰 단일 소스 적용

---

## 1. 결정 사항

### 1.1 왜 L4를 선택했는가

이전에 검토한 4단계 옵션 중 L4(Quarto 1.6+ `_brand.yml`)를 채택했다. 근거:

| 단계 | 비용 | 한계 |
|------|------|------|
| L1 metadata 옵션 | 작음 | HTML과의 토큰 분리 — 색상·폰트가 두 곳에서 관리됨 |
| L2 template-partials | 중간 | typst 측만 부분 커스텀 — HTML 일관성 미보장 |
| L3 커스텀 template.typ | 큼 | 자체 Typst 템플릿 유지보수 부담 |
| **L4 _brand.yml** | **중간** | **HTML/Typst/RevealJS 모두 같은 토큰 공유 (Quarto 표준 메커니즘)** |

핵심: **단일 소스 진실(SSOT)** — 인디고 팔레트와 폰트 정의가 한 파일에서 모든 출력 포맷으로 흐른다.

### 1.2 Brand가 다루는 것 vs SCSS가 다루는 것

| 항목 | Brand (`_brand.yml`) | SCSS (`scss/quarto-theme.scss`) |
|------|---------------------|--------------------------------|
| 색상 팔레트 | ✅ 인디고/뉴트럴/시맨틱 | (브랜드에서 받음) |
| 폰트 패밀리 | ✅ Pretendard / D2Coding | 폴백 스택 정의 |
| 한국어 행간 | ✅ 1.75 (base.line-height) | `word-break: keep-all` 등 세부 |
| Major Third 스케일 | — | ✅ `$font-size-xs..4xl` |
| Callout / 코드블록 스타일 | — | ✅ 색상·여백 규칙 |
| 한국어 자간(`letter-spacing`) | — | ✅ 큰 제목 -0.02em |

브랜드는 “토큰”, SCSS는 “규칙”을 담당하도록 책임 분리.

## 2. 파일 변경

### 2.1 추가

- `_brand.yml` (신규, 루트)
  - `color.palette`: indigo-50~900, neutral-50~900
  - `color.primary: indigo-600`, `secondary: neutral-600`, `link: indigo-600`
  - `color.success/info/warning/danger`: SCSS 토큰과 동일 값
  - `typography.base.family: "Pretendard"`, `line-height: 1.75`
  - `typography.headings.family: "Pretendard"`, `weight: 700`
  - `typography.monospace.family: "D2Coding ligature"`

### 2.2 정리

각 템플릿 `_quarto.yml`에서 brand로 이관된 항목을 제거하고, `brand: ../../_brand.yml`로 명시 지정:

```
templates/
├── report/_quarto.yml          # mainfont/monofont 제거, brand 추가
├── blog/_quarto.yml            # 위와 동일 + favicon 보존
├── book/_quarto.yml            # html + typst 동시
├── manuscript/_quarto.yml      # html + typst 동시
├── dashboard/_quarto.yml       # mainfont/monofont 제거
├── presentation/_quarto.yml    # revealjs theme 체인에 brand 추가
└── typst-pdf/_quarto.yml       # mainfont 제거, monofont만 D2Coding 명시
```

HTML 테마 체인 순서:
```yaml
theme:
  - cosmo                              # Bootstrap 기반
  - brand                              # 브랜드 토큰 주입
  - ../../scss/quarto-theme.scss       # 커스텀 규칙
```

## 3. 작업 중 발견된 함정

### 3.1 서브프로젝트는 루트 `_brand.yml`을 자동 탐색하지 않음

처음에는 루트에만 `_brand.yml`을 두면 모든 서브프로젝트가 인식할 거라 기대했으나, `templates/typst-pdf/index.typ` 출력에서 `#let brand-color = (:)` (빈 매핑)이 확인됨. 각 서브 `_quarto.yml`에 `brand: ../../_brand.yml`을 **명시적으로 지정**해야 했다.

### 3.2 Brand의 `logo`는 Typst에서 페이지 배경으로 자동 삽입됨

`_brand.yml`에 `logo.large: wordmark`를 두면 Typst가 표지/페이지 배경에 자동으로 SVG를 삽입한다. 하지만 Typst의 보안 모델은 **프로젝트 root 밖 파일을 읽지 못한다.** 서브프로젝트 root는 `templates/typst-pdf/`이므로 `../../assets/brand/...`는 차단됨.

→ **해결**: brand에서 `logo` 섹션 제거, HTML 측은 각 `_quarto.yml`에 `favicon: ../../assets/brand/favicon.svg`로 명시.

### 3.3 시스템 폰트명과 Typst가 인식하는 이름이 다름

`quarto typst fonts` 결과:
- ❌ "Pretendard Variable" — 미등록
- ✅ "Pretendard"
- ❌ "JetBrains Mono" — 미등록
- ✅ "D2Coding ligature"

→ **해결**: brand의 `typography.base.family`는 `"Pretendard"`로, `monospace.family`는 `"D2Coding ligature"`로 통일. HTML은 SCSS의 `$font-family-mono` 스택이 별도로 적용되어 영향 없음.

### 3.4 한국어 typst 행간

영문 기준의 Typst 기본 leading은 한글에 좁다. 모든 typst format에 `linestretch: 1.5` 명시.

## 4. 검증

```bash
quarto render templates/typst-pdf/index.qmd
# (W) brand.typography.base.size in rem units, changing to em
# [typst]: Compiling index.typ to index.pdf...DONE
# Output created: ../../docs/templates/typst-pdf/index.pdf
```

`index.typ` 중간 산출물에서 brand 토큰 주입 확인:

```typst
#let brand-color = (
  primary: rgb("#4f46e5"),       // indigo-600
  link:    rgb("#4f46e5"),
  indigo-500: rgb("#6366f1"),
  // ... 인디고/뉴트럴 팔레트 전체
)
#show heading: set text(font: ("Pretendard",), weight: 700, fill: rgb("#171717"))
codefont: ("D2Coding ligature",)
```

남은 경고:
- `rem → em` 단위 변환 — Typst가 rem 미지원으로 자동 변환 (무해)

## 5. 영향 범위

### 5.1 단일 소스가 된 토큰

이전: SCSS(`quarto-theme.scss`) + 각 `_quarto.yml`의 `mainfont`/`monofont`가 분산  
이후: `_brand.yml` 하나 — 변경 시 8개 템플릿에 동시 반영

### 5.2 호환성

- Quarto 1.6+ 필요 (현재 1.10.3 ✓)
- Typst 0.14+ 필요 (현재 0.14.2 ✓)
- 시스템 폰트 의존: Pretendard, D2Coding ligature (macOS 사전 설치 권장)

### 5.3 회귀 위험

- HTML 측: brand가 Bootstrap의 `$primary` 등을 덮어쓰므로 SCSS 규칙에서 `$primary` 사용 위치는 `_brand.yml`의 `color.primary`를 따른다. 인디고-600으로 통일됨.
- RevealJS 슬라이드 테마: `theme: [default, brand, quarto-revealjs.scss]` 순서로 brand가 중간에 끼므로 슬라이드 색상도 인디고로 적용됨.

## 6. 후속 과제

1. **블로그/대시보드/리포트 회귀 렌더 테스트** — 현 작업은 typst-pdf 위주로 검증. 다른 7개 템플릿도 한 번씩 렌더해 색상·폰트·로고 시각 확인 필요.
2. **`scss/quarto-theme.scss`의 토큰 중복 정리** — brand가 색상 변수를 주입하므로 SCSS의 `$color-primary-*`는 이제 backup용. 단계적으로 brand 변수 참조로 이관 가능.
3. **Pretendard·D2Coding 폰트 설치 안내 문서화** — README나 CLAUDE.md에 시스템 폰트 요구사항 추가.
